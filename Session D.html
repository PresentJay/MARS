HTTPS

HTTPS란
HTTPS는 Hypertext Transfer Protocol Secure의 약자로, 일반 HTTP 서비스에서 Web에 TLS/SSL 보안을 추가한 상태를 의미한다. 해당 상태가 되면 사이트 주소 표시줄에 작은 자물쇠 아이콘이 표시되며 http:// -> https://로 변경되어 주소가 표시된다.

HTTPS의 4가지 이점
1. 암호화 (Encryption)
교환되는 데이터를 암호화하여 도청장치 및 도청자로부터 안전하게 보호한다. 즉, 사용자가 사이트를 탐색하는 동안 어느 누구도 대화(하는 행동)를 듣지 못하고 여러 페이지에서 활동을 추적하거나 정보를 훔칠 수 없다.
2. 데이터무결성 (Data Intergrity)
데이터는 전송 중 의도적 또는 비의도적으로 감지 및 탐지되지 않고 수정되거나 손상 될 수 없다.
3. 인증 (Authentication)
사용자가 의도한 웹 사이트와 통신하고 있음이 증명된다. 이는 중간자 공격(man-in-the middle attacks)으로부터 보호하고 신뢰성을 구축하여 다른 비즈니스의 이점을 제공한다. 사용자의 입장에서는 웹사이트가 안전하다고 신뢰할 수 있는지를 확인하기 위해 자물쇠 아이콘을 찾는다고 할 수 있다.
4. SEO(Search Engine Otimization) 혜택 제공
HTTPS 사이트는 검색성이 높아지게 된다. 검색 엔진이 최적화(SEO)된다.

HTTPS를 사용해야하는 이유
getUserMedia()를 통한 오디오 녹음 또는 사진 촬영, 서비스 워커를 통한 오프라인 앱 경험 활성화, 또는 Progressive Web App 빌드를 비롯한 강력한 새 플랫폼 기능은 사용자가 명시적으로 허가해야 실행됩니다. geolocation API 등과 같은 대부분의 이전 API도 실행하려면 허락이 필요하도록 업데이트되는 중입니다. HTTPS는 이러한 새 기능과 업데이트된 API에 대한 권한 워크플로의 주요 구성 요소입니다.
